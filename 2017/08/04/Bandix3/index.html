从0到1的Bandix（三）- vue.use

前端 实验室 vue

我不指望Bandix有多牛逼，跟Syuanpi一个态度，我只是想给自己做作业

其实，完整的函数是Vue.use(plugin)，在中文文档上是这么说的

安装 Vue.js 插件。如果插件是一个对象，必须提供install方法。如果插件是一个函数，它会被作为 install 方法。install 方法将被作为 Vue 的参数调用。
当 install 方法被同一个插件多次调用，插件将只会被安装一次。

所以实际上，我们使用一个库的时候，组件库本身是把自己当做一个插件插入到项目中，而不是一个什么大头鬼。

Vue use

而且官方也提供了几个例子

添加全局方法或者属性，如:vue-custom-element
添加全局资源：指令/过滤器/过渡等，如vue-touch
通过全局 mixin方法添加一些组件选项，如:vuex
添加 Vue 实例方法，通过把它们添加到Vue.prototype上实现。
一个库，提供自己的 API，同时提供上面提到的一个或多个功能，如vue-router

要用Vue.use()的话，use的这个东西里面必须有一个公开的install方法，方法要两个参数，第一个就是Vue，第二个是个可选的option

所以，既然方法已经把Vue实例传进去了，那么就意味着，vue的全局函数都可以在这个方法里面调用，什么component, extend, mixin等。

那么库的入口就好解决了

入口是解决了，那么如果发布怎么办？

一切npm包都是靠package.json啊这个很早就有提到，不管发布还是管理，都要靠这个json文件。如果你的入口js文件写好了，入口js文件的install方法确定是妥的，那么在package.json中，main节点大胆写进去就行了。这样届时从npm下载的话，安装完，该怎么import怎么import，然后use，完事儿！
