从0到1的Bandix（二）- component & extend

前端 实验室 vue

如果没玩过Nintendo Switch的话，就千万不要去接触这个东西，这东西让人鬼迷心窍的！

在vue中，可能无论写一个什么，就算只是单纯的一个vue插件，也会用到vue component。因为你搞一个*.vue就已经是了啊（逃

一切还是以探索为主，我只是想把看到的东西按照自己理解记录一下。关于这部分的内容，在Vue组件这里有中文文档，在Vue.extend有关于extend的内容。

现在大多数组件库的话，都是Vue.use()来使用，那么为什么use就能用了？到底是在u些什么鬼东西。

Vue component

简单来说，就是

所以就算看一些组件库的话，在入口js文件看到最多的就是这玩意儿。这是一个全局引用组件的方法。一次引用了，在项目里哪个地方都能使用了。

当然还有局部引用方法，这个就常见了，你在每个vue文件里，<script>区块里写的那个components: {}不就是这个作用！

所以按照我的理解，Vue.component主要还是一个声明，一个注册组件的作用，毕竟你写组件的话都直接用*.vue了，搞这东西来写不是很蛋疼。

Vue extend

当然没叫你这么写！

所以extend来说，这就是一个类似于*.vue但可以不专门弄一个文件来写的，一个类构造器的东西。或者说是组件构造器吧，另一种写组件的姿势。它的后缀就可以不是vue了，而是一个纯js文件。

这种我觉得，比较适合那种模板内容不多，但是逻辑或者功能又特别复杂的东西。比如插件之类的开发，因为就一个js文件，看起来也很舒服。而且它是以扩展的形式插入进去的，当然可以全局扩展也可以局部扩展，跟component是一样的。

不过这种写法，让我觉得可以用jsx或者typescript来皮一下是怎么回事……不过typescript确实还是大多靠这种方式来实现，很好玩我很喜欢玩。不知道现在有没有更好的方法。而且jsx有官方插件可以用了，加上render函数。我怎么觉得有点变味……
