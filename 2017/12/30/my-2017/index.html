    扣肉小记

    我的2017

    我本来是最反感写年终总结的那一个，我最讨厌的就是形式化的东西。

    但感觉今年还是挺有意义的，学着去接触了很多东西，所以我还是愿意记录下所有第一次。

    -2017

    第一次自我驱动做了NodeJS尝试

    说实话这并不属于2017，属于2016 mid，但这是我真正愿意触摸“新JS时代”的第一步。

    在尝试接触hexo的时候，一次偶然的机会我接触到了使用yilia的博客，其中有一篇博客是讲“爬取慕课网课程”的文章，用Java实现。那个时候我想我也实现一个好了，不是说NodeJS现在很厉害很火么。于是我就以node脚本的形式去做了，用着十年前的前端JS的能力。

    抓取地址、用request爬虫、cheerio分析dom，当时觉得自己很牛逼了，同时坚定了继续读JavaScript的想法。

    所以我的2017，准确来说应该从这个时候开始的。

    第一次开源

    从这之后，我写完要发布啊，要开源啊。“KPA思想”也从这个时候开始生根发芽。所以学习了git，注册了github（没错我就用了1年），把这个几K的东西放上去，所以这也是我的第一个项目，只不过凉了。

    刚刚不是提到hexo嘛，没错后面就开始关于hexo的开发——做主题。

    目标很明确，我接触hexo不是来写文章的，而是来写主题装逼的。这个时候我依然还是老一套的前端思想（03年接触的全静态网站开发，08年嫌弃JS菜，溜了）。

    所以在早期的文章应该还有提到，写了一个主题Rousy。从代码到设计风格完全就是十年前的东西，如果不用div + css3的话估计还会被以为是远古产物。那个时候去跟已经学了很久前端的同学 @Ahonn 说了这件事，人家就一句评价：

    好稚嫩；
    所以你也算是入了前端的坑？

    没错，以我的性格，立刻回了一句“我一直在前端坑里面啊”。回完之后马上意识到，我凉了。

    2017

    所以到这里才是关于今年的事情。

    编程

    在这之后又开发了一个主题Blank，想法就是要大部分留空，只保留想要的东西，而且使用大量的交互动画。页面甚至只剩下标题，只有鼠标悬浮才能看到概要或者整篇文章。但因为设计思路本身存在问题，所以弃了。

    由于意识到现在前端已经今非昔比了，加上前面两次积累了许多经验。于是今年第二个“KPA”产物就是现在正在使用的 Nlvi ，第一个版本“纸”。

    Nlvi的出生并不是纯洁的，它一开始只是想用来证明自己。拿着它去找工作。所以我除了提交到 hexo theme 并没有做其他的宣传。直到过了一整天，在项目看到第一个star的时候我才知道，我错了。

    我开始修正我的想法，毕竟nlvi也是love类型的。所以经历了一年的迭代，有了新的版本“彩”。

    然而“彩”的初衷是看到了issue有一条评论：迫切需要颜色。

    所以如今 Nlvi 相比年初，它更完善了。也保留了最初的设计融合了颜色的内容，多了一种风格。让我感动的是，目前已经至少有92人对它的肯定。主题还有两个老哥一起维护，虽然一次瞎眼操作让他们从contributor消失了，但我还是记住他们：ruinshe, lostinlight。

    开始逐渐适应了现在的网页，迎合现在比较清晰的设计。所以把一些常用的CSS3动画打包成库，加上一些自己的小想法，成为我的第一个动画库——syuanpi.css。但是它并没有什么好讲的，但它同样有意义——第一次接受了知乎用户的怜悯。

    学习

    今年学习可就多了，意识到自己离“新前端”至少得有十万八千里。所以我开始专注学习JavaScript，在这之前还尝试过angular和TypeScript呢，也是不了了之。

    不过我并没有直接去看书，而是去看别人的代码。去学现在大家都是怎么写怎么对待这门学科的。直到后面理解了虽然你是能做事了，还是少不了看书，所以开始读《红宝书》、《DOM艺术》。

    高中水技术的时候，已经从一位我很敬仰的工程师了解到，这个世界分编程语言的话只分成两种：指令式编程，函数式编程。还有一些适应性强的语言，它们叫多范式。例如C系都是指令式，JavaScript是函数式。

    从这之后，“函数式编程”只是我用来装b的词汇。但从今年开始，学js同时也学函数式编程思想。

    真正开始“拿函数式当枪使”的时候是 @Ahonn 吹了一次ClojureScript，从这开始我从小打小闹直接进入疯狂沉迷状态。说起来我的Hyper好像要凉了…每次用TS都感觉手感不太对的样子…

    一直使用现代编辑器的我看到别人用vim、emacs都会在心里嘲讽他，没错今年轮到我当sb了，开始使用vim8和spacemacs。vim还在慢慢配置的状态，平时多数是 linux + macos，vim多数在linux那边使用的。现在mac的质量和手感越来越差，也是找个借口让自己适应windows + linux吧。

    有时候还是会不适应spacemacs，所以保留了vscode。结果一年下来用得最多的依然是vscode。

    还是跟往常一样，学习没规律，只要不上班就哐哐开始搞。看到 @Ahonn 使用“番茄工作法”规划时间，我想2018年也尝试一下，取长补短嘛。

    工作

    创业划水之后，好像也挺无聊的然后第一次去上班。因为起步比别人慢，所以基本上错过了所有校招，加上学历问题，最后面试了两家外包公司，去了一家交通方便的。

    基本上就是各种不适应咯，并不是上班不适应，而是编程范式不适应。过于稳定的技术意味着LOW，还有各种“奇妙”的问题解决思维。也见识过各种金主爸爸的无理取闹，使得开发从完整系统到拆散系统；见识过设计稿毫无章法不给规范说明却要求严格把控细节；见识过严格要求遵守规范却没有规范文件，想到什么规范什么。

    嘛，没坏处，至少比其他人多体会到更多的东西。至少我现在可以站着跟你说：你能够用微信小程序开发出7个路由的应用吗？你开发过20个页面的小程序吗？

    游戏/读书

    单片机，树莓派，还有上面说的，每天都是以搞为生，一年接触一堆语言和设计模式。当然游戏依然是我的主要放松方式。

    今年也没打通过什么游戏，年初买了Nintendo Switch，只打通了《马里奥·奥德赛》，其他的游戏基本上也不算通，毕竟有意思的内容都是网络部分。现在在玩《异度之刃2》，毕竟是这个月的游戏，一个月通关对我来说真的不可能。

    社交元素，玩起了《王者荣耀》，但我并不喜欢玩手机游戏，为了搭上话才去玩。一天两把，有一天没一天的，三个赛季两个白金一个钻石。

    相比去年今年的游戏时间可以说少了很多，主要都花时间在搞开发语言上了。

    读书的话，除了一些新闻、报纸杂志，其他文学类的书我也不是很能读得下去，所以看了比较多的技术类。读过一两本也是题材得非常感兴趣的，翻了几页《草木缘情》，在多看读了两本东野圭吾的小说。然后大多数时间都在设计模式，数据结构。

    人在江湖混，迟早是要还的。

    计划

    没有计划！

    这就是真实的我，从来不会先给自己夸下海口。计划永远赶不上变化，只要时间不是浪费的就好了。哪怕玩游戏，游戏能带出来的内容也是相当多的。

    一整年 @Ahonn 隐隐约约给的帮助还是相当大的，很感谢。

    当年在孵化池里听到师姐说你一早的打算就是毕业前能去大厂，确实如愿以偿啦。恭喜同时也很羡慕。

    我呢，我还需要再凉两年吧！
