    这是一个失败者的总结

    这个年终总结的时间是错误的，真正完成的那天远晚于当天。

    我本来还是跟去年一样已经不想写的，但坐着也无事，总结一下今年的事情。

    这篇文章诞生于支离破碎的主题中，因为博客主题正在改造，很长时间没更新也是因为改动太大完全收拾不开..

    一事无成

    今年打算完成的内容可以说没有一件完成的：TFD(The Fking Design)，和以此为基础的Nlvi-禅、syuanpi、Sona。以及胡乱开坑的双兔Bunny、Rabbit。还有两个移动应用…… Rabbit可以说已经在公司偷偷用上了。

    TFD 还是有出的，按照规范出一个Sketch，只不过设计这东西实在太费时间了，可以想象一下，你7点打开Sketch，开一个新组件，调整两条线条，感觉也就过了半小时，抬头，已经12点…还有大量内容停留在脑子里没实现，这东西还是要尽快，毕竟如果你没天天去回想的话是会忘记的，比如syuanpi的新效果…

    前段时间把syuanpi.css的主页改成新风格，调整了时间栅格和一些过渡不自然，但是很多想法并没有实现。纠结于WAAPI的 JS 版本更是开了个头完全没实现…估计弃坑，正在尝试React版本的场景动画。

    新的Nlvi在写，还是会改布局改风格，逻辑代码部分也会改，现用的搜索和标签云会改掉，归档页打算用瀑布流，场景动画暂时不会去想吧，毕竟之前尝试过结合pjax的场景动画，拉闸…

    主要还是换了个新环境，还以为能跟以前一样浪，实际精力真实跟不上，白天跟孤儿斗智斗勇晚上真的抬不开眼，连 switch 都不是很愿意拿。

    还是想吐槽自己，今年 GitHub 格子实在太空…

    讲点有意思的

    还是有高兴事的

    今年换了新环境，所以现在半个战场转移到这边，用爱发电（不算KPI）做了几个工具：CLI 和 VSC 插件都有，甚至改了一个超小的状态管理库。而且原本用的是vue技术栈，今年刚好一个机会提出转react，然后整个团队在年底全部转完了。这我还是挺高兴的，react真是一个可以依照思路从头写到尾的框架，而且 vim 也能很好找到支持和补全（装b更顺利了）。

    买了个 kindle，看完很多书很多小说，可以说充实了通勤时间，多次公交坐过站，看非技术和小说还是很上瘾的，但是看技术类真的很容易犯困…所以今年应该调整书籍策略，不要在路上看过于硬核的书，容易犯困又没成效。后来为了支持老罗买了一个坚果Pro2S，在此劝诫能看到这篇文章的人，别买垃圾屏幕的手机，我现在回想起我眼睛废掉的原因大概率是用锤子低背光在车上看东西…这手机挺好的，就是背光不要调低，屏闪真的很伤眼。飞行员视力从今年开始已然不存在…

    说是这么说，在车上还是看完了《Haskell 趣学指南》，看的时候比较兴奋，下车到电脑边实践半小时，断断续续看完。Haskell 确实对今年写代码的思路提升了很多，可以说入门就得到了效果，但还是要多玩，很多核心的思想还没掌握到。

    很重要的一点，终于意识到TypeScript的优势了，其实回想起一年前或者两年前的自己，根本不能算上一个程序员，「胶水工程师」都算不上。TS0.5时代就接触到TS，到今天才认识，惭愧不惭愧。

    期望

    • 好好活着，特别现实的想法

    • 少跟孤儿争论一些有的没的

    • 少嫌队友菜，各种意义上的队友…或许太硬核的技术是不必要的

    • 精准打击，不要再做机器学习了！

    • 调整看书策略，总是犯困也不是个事

    • TFD好好做完，还有几个坑依次填完

    • 一些操作还是抽象出工具吧，不要再拿手速自己骗自己了…
